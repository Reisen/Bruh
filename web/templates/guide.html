Code Execution

Walnut can execute code in several languages. The general syntax for running a snippet of code is written [lang], [code]. For example, running a snippet of haskell can be written h, 1 + 1. Currently the following languages should work:

  • c, [code]Clang C++
  • w, [code]GCC C++
  • h, [code]Haskell
  • s, [code]Scala
  • r, [code]Rust
  • j, [code]Javascript

Stop Walnut Being Annoying (Not Yet Implemented)

If you have at least half-op in the channel in question, you can toggle commands on and off completely in the channel using .enable and .disable commands. Entire plugins can be disabled for the channel using the .toggle command.

If you aren't a channel admin, you can still exclude yourself from certain behaviours that walnut performs. Walnut runs regular expressions and sinks on every message it sees. You can exclude yourself from these by running .exclude [exclusion]. The following exclusions are possible:

  • butt_sink Your messages won't trigger random buttifications.
  • stat_sink Your messages aren't counted for statistics.
  • regular_sink Your messages won't trigger regular expressions.
  • recall_match Your messages starting with ? won't lookup factoids.
  • karma_match Your messages won't be parsed for karma.
  • youtube_match Youtube links you send won't have their information printed.
